# Contributing to EPL Match Tracker

Thank you for your interest in contributing to EPL Match Tracker, a DSCI-532 Group 6 project!

## How to Contribute

- Fork the repository and create your own branch for new features or bugfixes from the `main` branch.
- Make your changes in the feature branch.
- All code changes should be submitted via pull request and reviewed by a team member before merging.
- Follow the project's code of conduct.
- Use GitHub Issues for task assignment, bug reporting, and technical discussions.

## Project Workflow

1. **Branch Creation**: Create a branch for each feature or fix from the `main` branch.
2. **Pull Requests**: Submit a pull request for your changes. At least one other teammate must review the pull request before merging.
3. **Code Review**: Incorporate feedback from reviewers to improve code quality and project understanding.
4. **Merging**: Once approved, the pull request can be merged into `main`.

## Example Contributions

You can contribute in the following ways:
- Report bugs or issues via GitHub Issues.
- Fix bugs or implement features from open Issues.
- Write or improve documentation in README.md, team contract, or data files.
- Submit feedback via GitHub Issues.

## Contact

For questions or suggestions, reach out through GitHub Issues or via the team Slack channel.
